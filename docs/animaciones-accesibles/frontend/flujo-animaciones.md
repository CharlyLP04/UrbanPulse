# Identificación de Flujos con Valor Real – Frontend

## Objetivo

Identificar flujos de interacción donde la animación aporte valor real a la experiencia del usuario dentro de UrbanPulse.

---

## 1. Apertura del formulario de reporte

**Contexto:**  
El usuario presiona el botón "Crear Reporte Ahora" desde la página principal.

**Problema UX:**  
Un cambio abrupto de pantalla puede generar desorientación.

**Valor real de la animación:**  
Una transición suave (fade + desplazamiento leve) comunica continuidad visual y confirma que la acción fue reconocida por el sistema.

**Impacto en experiencia:**  
- Reduce sensación de salto brusco.
- Mejora percepción de fluidez.
- Refuerza respuesta inmediata del sistema.

---

## 2. Retroalimentación al enviar un reporte

**Contexto:**  
El usuario envía un formulario con información.

**Problema UX:**  
Si no hay retroalimentación clara, el usuario puede pensar que el sistema no respondió.

**Valor real de la animación:**  
Un estado visual de carga seguido por transición a éxito o error comunica claramente el resultado del proceso.

**Impacto en experiencia:**  
- Reduce ansiedad.
- Aumenta confianza en el sistema.
- Clarifica el estado actual (procesando, éxito o error).

---

## 3. Transición de estado: cargando / error / éxito

**Contexto:**  
El sistema cambia entre diferentes estados visuales después de una acción.

**Problema UX:**  
Cambios instantáneos sin transición pueden generar confusión.

**Valor real de la animación:**  
Transiciones suaves entre estados ayudan al usuario a entender que el sistema evolucionó de un estado a otro.

**Impacto en experiencia:**  
- Mejora claridad del flujo.
- Refuerza comprensión del resultado.
- Mantiene coherencia visual.
